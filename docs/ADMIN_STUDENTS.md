# مدیریت دانش‌آموزان

## داشبورد و فهرست

- داشبورد تعداد کل، تأییدشده و در انتظار را از endpoint صفحه‌دار استخراج می‌کند.
- فهرست از `/admin/students/with-stats` استفاده می‌کند و محدودیت endpoint ساده ۱۰۰ رکوردی را ندارد.
- صفحه و وضعیت تأیید در `page` و `approved` URL نگه‌داری می‌شوند.
- هر ترکیب filter/page کلید cache مستقل دارد و هنگام تعویض صفحه داده قبلی تا دریافت پاسخ حفظ می‌شود.
- loading، empty، error، تعداد کل و کنترل صفحه‌بندی پوشش داده شده‌اند.

## پرونده دانش‌آموز

- query اصلی پروفایل و queryهای exams، mistakes، statistics و performance مستقل‌اند.
- خطای یک تب داده سایر تب‌ها را از بین نمی‌برد.
- مشخصات مجاز شامل نام، نام خانوادگی، شهر، مدرسه و رشته inline ویرایش می‌شوند.
- وضعیت تأیید، لغو تأیید و حذف با mutation اختصاصی و invalidation شاخه مدیر انجام می‌شود.
- حذف پرونده و لغو تأیید confirm صریح دارند؛ عملیات destructive optimistic نیست.

## گزارش عملکرد

- مدیر گزارش جدید با تاریخ جلالی، برنامه مطالعاتی، یادداشت و پیوست ثبت می‌کند.
- ویرایش notes، study plan و files مطابق محدودیت endpoint update انجام می‌شود؛ تاریخ در ویرایش غیرفعال است.
- فایل‌ها پیش از ذخیره با `/upload/multiple?type=document` آپلود و URLها به‌صورت JSON string ارسال می‌شوند.
- سقف تعداد و حجم هر فایل از env خوانده و پیش از upload کنترل می‌شود.
- پیوست‌های قبلی در ویرایش به URL خام backend نگه‌داری می‌شوند و URL gateway در دیتابیس ذخیره نمی‌شود.

## مرز امنیتی و cache

- layout مدیریتی پیش از render، session و `role=admin` را در Server Component کنترل می‌کند.
- backend نیز تمام endpointهای `/admin` را با middleware نقش محافظت می‌کند.
- queryهای مدیر همگی زیر prefix `admin` هستند و logout کل QueryClient را پاک می‌کند.
- هیچ درخواست مدیریتی از صفحه یا layout دانش‌آموز اجرا نمی‌شود.
