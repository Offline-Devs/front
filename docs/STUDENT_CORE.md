# هسته پنل دانش‌آموز

## داشبورد

- summary از `GET /students/dashboard` با stale time کوتاه دریافت می‌شود.
- وضعیت تأیید پروفایل و وجود برنامه مطالعاتی به‌صورت صریح نمایش داده می‌شوند.
- آزمون‌ها و اشتباهات اخیر همراه با مسیر مستقیم مشاهده/ثبت در دسترس‌اند.
- خطای مستقل دفترچه اشتباهات باعث از دست رفتن summary اصلی نمی‌شود.

## آزمون‌ها

- فهرست، جزئیات، ایجاد، ویرایش و حذف به endpointهای `/exams` متصل هستند.
- رشته و درس‌ها از تنظیمات عمومی backend دریافت و یک روز cache می‌شوند.
- فرم با `useFieldArray` هر تعداد درس را مدیریت می‌کند.
- برای هر درس دو invariant کنترل می‌شود: `correct + wrong = answered` و `answered + blank = total_questions`.
- درس تکراری و تاریخ جلالی نامعتبر پیش از درخواست رد می‌شوند.
- `total_subjects` همیشه از طول آرایه محاسبه می‌شود و قابل دستکاری مستقل نیست.
- در ویرایش، هشدار replace کامل subjects مطابق رفتار backend نمایش داده می‌شود.

## دفترچه اشتباهات

- فهرست، جست‌وجو، ثبت، ویرایش inline و حذف با تأیید پیاده‌سازی شده‌اند.
- آزمون و درس فقط از query آزمون‌های کاربر جاری انتخاب می‌شوند؛ شناسه آزاد دریافت نمی‌شود.
- ارتباط با آزمون/درس اختیاری است و مقدار خالی پیش از ارسال به `undefined` تبدیل می‌شود.
- شماره سؤال مثبت و دسته‌بندی غیرخالی الزامی هستند.

## پروفایل

- داده پروفایل در server دریافت و به فرم مشترک ویرایش منتقل می‌شود.
- وضعیت تأیید فقط نمایش داده می‌شود و دانش‌آموز امکان تغییر آن را ندارد.
- ذخیره موفق cache پروفایل و dashboard را invalidate می‌کند و پیام موفقیت نشان می‌دهد.
- آپلود و تعویض عکس از همان pipeline امن فاز onboarding استفاده می‌کند.

## cache و mutation

- کلیدهای list و detail آزمون مستقل‌اند.
- create/update/delete فقط شاخه‌های exams، dashboard و statistics را invalidate می‌کنند.
- mutation اشتباه فقط mistakes، dashboard و statistics را invalidate می‌کند.
- حذف‌ها optimistic نیستند؛ چون عملیات destructive و برگشت‌ناپذیرند، ابتدا confirm و سپس پاسخ قطعی server اعمال می‌شود.
