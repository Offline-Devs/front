# احراز هویت و onboarding

## جریان ورود

1. شماره موبایل با ارقام فارسی، عربی یا لاتین دریافت و به قالب `+989xxxxxxxxx` تبدیل می‌شود.
2. درخواست OTP فقط از BFF هم‌مبدأ `/api/auth/request-otp` عبور می‌کند.
3. شماره تا پایان جریان در `sessionStorage` همان tab نگه‌داری می‌شود و در URL یا log قرار نمی‌گیرد.
4. OTP شش‌رقمی از تایپ، paste و one-time-code مرورگر پشتیبانی می‌کند.
5. بعد از تأیید، tokenها فقط در cookie رمزگذاری‌شده HttpOnly ذخیره می‌شوند و browser صرفاً اطلاعات کاربر را دریافت می‌کند.

## هدایت بعد از ورود

- مدیر مستقیماً به `/admin` هدایت می‌شود.
- برای دانش‌آموز، `GET /students/profile` اجرا می‌شود.
- پاسخ ۲۰۰ به `/dashboard` و پاسخ ۴۰۴ به `/complete-profile` هدایت می‌شود.
- پاسخ ۴۰۳ کاربر غیرفعال و پاسخ ۴۰۱ کد اشتباه/منقضی با پیام فارسی کنترل‌شده نمایش داده می‌شوند.
- در خطای شبکه هنگام تشخیص پروفایل، کاربر در همان فرم باقی می‌ماند تا هدایت اشتباه رخ ندهد.

## OTP و rate limit

- زمان resend از `NEXT_PUBLIC_OTP_RESEND_SECONDS` خوانده می‌شود.
- resend موفق شمارنده و ورودی‌ها را reset می‌کند.
- پاسخ ۴۲۹ شمارنده را دوباره فعال می‌کند و مانع ارسال پشت‌سرهم می‌شود.
- OTP بازگشتی provider آزمایشی فقط وقتی backend صریحاً آن را برگرداند در UI نشان داده می‌شود.

## تکمیل پروفایل

- نام، نام خانوادگی، شهر، مدرسه و رشته validation می‌شوند.
- تاریخ جلالی با ارقام فارسی یا لاتین پذیرفته و پیش از ارسال به `YYYY/MM/DD` تبدیل می‌شود.
- رشته‌ها با cache یک‌روزه از endpoint عمومی majors دریافت می‌شوند و در خطای موقت fallback قابل ورود دارند.
- عکس ابتدا از endpoint احراز‌شده upload ارسال و URL بازگشتی در profile ذخیره می‌شود.
- نوع، حجم، preview، حذف و خطای upload در client کنترل می‌شوند.
- پس از ذخیره، cache پروفایل و dashboard دقیقاً invalidation می‌شود.

## فیلدهای پویا

renderer انواع `text`، `number`، `select`، `checkbox` و `date`، options امن و required runtime را پشتیبانی می‌کند. قرارداد فعلی backend فقط endpoint مدیریتی `GET /admin/dynamic-fields` دارد؛ بنابراین تعریف‌های entity دانش‌آموز هنوز برای session دانش‌آموز قابل دریافت نیستند. فرم بدون hard-code کردن تعریف‌ها آماده دریافت آن‌هاست و تا اضافه‌شدن endpoint خواندنی برای دانش‌آموز، `dynamic_fields` را به‌صورت خالی و معتبر ارسال می‌کند.

## تست و امنیت

- نرمال‌سازی شماره و ارقام، انتخاب مقصد بر اساس role/profile و validation تاریخ جلالی تست شده‌اند.
- paste کد فارسی و هدایت مدیر در تست component پوشش داده شده است.
- صفحات ورود و onboarding از index شدن توسط crawler جلوگیری می‌کنند.
