# آمار و عملکرد دانش‌آموز

## دریافت و فیلتر داده

- داده آماری از `GET /students/statistics?from=&to=` و گزارش‌ها از `GET /students/performance` دریافت می‌شوند.
- بازه جلالی با ارقام فارسی/لاتین پذیرفته و به `YYYY/MM/DD` تبدیل می‌شود.
- ترتیب بازه پیش از درخواست کنترل می‌شود و `from`/`to` در URL منبع حقیقت query هستند.
- query key شامل فیلترهاست؛ هر بازه cache مستقل دارد و stale time آمار ۶۰ ثانیه است.

## نمایش آمار

- کارت‌ها تعداد آزمون، میانگین و مجموع اشتباهات دسته‌بندی‌شده را نشان می‌دهند.
- نمودار خطی روند آزمون‌ها، نمودار میله‌ای درس‌ها و نمودار حلقه‌ای دلایل اشتباه پیاده‌سازی شده‌اند.
- داده نمودارها با `useMemo` مرتب/گرد می‌شوند و render دوباره محاسبه غیرضروری ندارد.
- هر نمودار روند/درس fallback جدولی بازشدنی دارد و دلایل اشتباه همیشه جدول متنی دارند.
- نمودارها label دسترس‌پذیر دارند و اطلاعات فقط با رنگ منتقل نمی‌شوند.

## بهینه‌سازی bundle

- سه component وابسته به Recharts با `next/dynamic` و `ssr: false` بارگذاری می‌شوند.
- صفحات dashboard، آزمون و عملکرد کتابخانه نمودار را در chunk اولیه دریافت نمی‌کنند.
- loading chart مستقل است و layout هنگام دانلود chunk جابه‌جا نمی‌شود.

## timeline عملکرد

- گزارش‌ها با ترتیب backend در timeline RTL نمایش داده می‌شوند.
- برنامه مطالعاتی و یادداشت با حفظ line break و بدون HTML خام render می‌شوند.
- `files` توسط mapper با JSON parse دارای fallback امن به آرایه تبدیل می‌شود.
- URL نسبی پیوست از same-origin gateway عبور می‌کند و لینک خارجی با `noopener noreferrer` باز می‌شود.
- خطا، loading و empty state مستقل پوشش داده شده‌اند.
